### Context
* 개인 프로젝트 및 n8n 워크플로우 자동화를 위한 비용 효율적인 클라우드 환경 구축이 필요함.
* Google Cloud Platform(GCP)의 Always Free(항상 무료) 티어를 활용하여 `e2-micro` 인스턴스를 프로비저닝함.
* 무료 제공 리전 제한 및 월별 리소스 한도를 정확히 준수하여 의도치 않은 추가 과금을 방지하는 것이 핵심 목표임.

### Core
* 무료 티어가 적용되는 주요 설정 조건
  * 인스턴스 타입: `e2-micro`
  * 지원 리전: `us-central1` (아이오와), `us-east1` (사우스캐롤라이나), `us-west1` (오리건) 중 택일
  * 부팅 디스크: 30GB 표준 영구 디스크 (Standard Persistent Disk)
  * 네트워크: 북미 외 아웃바운드 트래픽 월 1GB 무료 (프리미엄 네트워크 티어는 과금될 수 있으므로 표준 티어 권장)
* gcloud CLI를 통한 무료 인스턴스 생성 스니펫
```bash
gcloud compute instances create free-n8n-server \
    --project=my-gcp-project-id \
    --zone=us-central1-a \
    --machine-type=e2-micro \
    --network-interface=network-tier=STANDARD \
    --create-disk=auto-delete=yes,boot=yes,image-project=debian-cloud,image-family=debian-11,size=30GB,type=pd-standard
```

### Insight
* `e2-micro` 인스턴스는 2개의 vCPU(버스트 가능)와 1GB RAM을 제공하여 경량화된 Python 스크립트 실행이나 단일 n8n 도커(Docker) 컨테이너 운영 등 소규모 에이전트 작업에 적합함.
* 설정 과정에서 디스크 타입을 기본값인 `pd-balanced`나 `pd-ssd`로 설정할 경우, 무료 티어 기준에 부합하지 않아 과금이 발생할 수 있으므로 반드시 `pd-standard`로 수동 지정해야 함.
* 기본 제공 메모리가 1GB로 매우 적은 편이므로, 서버 생성 직후 스왑(Swap) 메모리를 최소 1GB 이상 할당하여 시스템이 메모리 부족(OOM) 오류로 중단되는 현상을 예방하는 것을 권장함.
