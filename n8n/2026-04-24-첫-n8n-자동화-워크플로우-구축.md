### Context
* 디스코드(Discord) 채널에 메모를 입력하면, 자동으로 구조화된 문서로 변환되어 깃허브(GitHub)에 업로드되는 워크플로우를 구축함.
* 사용자가 파편화된 메시지를 남기더라도 AI 에이전트(Agent)가 TIL(Today I Learned) 형식으로 변환하고, 레포지토리 인덱싱과 완료 알림까지 원스톱으로 처리하는 자동화 환경을 구성하는 것이 목표임.

### Core
* 워크플로우 파이프라인의 핵심 노드 구성 및 데이터 흐름은 다음과 같음.

```json
{
  "nodes": [
    {
      "node": "Discord Trigger",
      "action": "메시지 수신(Message Created) 감지 및 텍스트 페이로드 추출"
    },
    {
      "node": "AI Agent",
      "action": "프롬프트를 바탕으로 수신된 메시지를 TIL 마크다운 포맷으로 가공"
    },
    {
      "node": "GitHub Push",
      "action": "생성된 마크다운 데이터를 기반으로 Repository에 새로운 파일을 Commit & Push"
    },
    {
      "node": "GitHub Update",
      "action": "기존 README.md 파일을 불러와 새 인덱스 라인을 추가(Append)한 뒤 재 Commit"
    },
    {
      "node": "Discord Notification",
      "action": "성공 여부 및 GitHub 링크를 원본 채널에 완료 메시지로 발신"
    }
  ]
}
```

### Insight
* n8n 워크플로우를 처음 다루어 보았으나, 본질적인 작동 원리는 각 노드(Node) 간의 입력(Input) 및 출력(Output) 데이터 페이로드(Payload)를 매핑하는 것임을 깨달음.
* 개별 노드가 요구하는 파라미터가 무엇인지, 이전 노드에서 어떤 JSON 결과값을 내보내는지만 명확하게 파악하면 아무리 복잡한 파이프라인도 유연하게 연결할 수 있음.
* 진입 장벽을 낮추고 워크플로우 자동화의 핵심 개념을 알려준 후츠릿(Hootsrit)에게 큰 감사를 표함.
