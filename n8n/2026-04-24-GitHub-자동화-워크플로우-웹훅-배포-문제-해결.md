### Context
* 로컬 환경에서 정상 작동하던 GitHub 자동화 워크플로우를 프로덕션 서버에 배포한 후 트리거가 작동하지 않는 문제가 발생함.
* n8n에서 GitHub Trigger 노드를 설정했으나, GitHub 저장소의 웹훅(Webhook) 페이로드가 서버로 정상적으로 전달되지 않아 워크플로우가 실행되지 않음.
* 로컬에서는 n8n에서 제공하는 기본 터널링을 통해 접근이 가능했지만, 서버 배포 환경에서는 공인된 외부 접속 엔드포인트 URL 처리에 문제가 있음을 확인함.

### Core
* n8n을 도커(Docker) 컨테이너 기반으로 서버에 배포할 때, 호스트 서버의 도메인과 n8n이 인식하는 웹훅 URL을 일치시켜야 함.
* `docker-compose.yml` 파일 내에 `WEBHOOK_URL` 환경 변수(Environment Variable)를 명시적으로 설정하여 이 문제를 해결함.

```yaml
version: '3.8'

services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=n8n.my-domain.com
      - N8N_PORT=5678
      - N8N_PROTOCOL=https
      - NODE_ENV=production
      - WEBHOOK_URL=https://n8n.my-domain.com/
      - GENERIC_TIMEZONE=Asia/Seoul
    volumes:
      - n8n_data:/home/node/.n8n
```

### Insight
* n8n은 기본적으로 컨테이너 내부 IP 및 포트를 기반으로 웹훅 주소를 자동 생성함.
* Nginx와 같은 역방향 프록시(Reverse Proxy) 뒤에 n8n을 배치하거나 자체 도메인을 연결하는 경우, GitHub이 도달해야 하는 정확한 주소를 `WEBHOOK_URL`에 명시하지 않으면 노드에서 잘못된 콜백 URL이 생성됨.
* 외부 서비스와 연동되는 도달성(Reachability) 검증 시, 언제나 n8n UI의 'Webhook URLs' 메뉴에서 프로덕션 URL이 설정한 도메인과 완벽히 일치하는지 교차 검증하는 과정이 필수적임.
